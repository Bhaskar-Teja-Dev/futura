1. Why this stack

The original spec listed Google Stitch as the frontend. Google Stitch is not a publicly available product — the closest match in Google's ecosystem is Firebase App Hosting or Firebase Studio. Rather than build on an undocumented surface, this document replaces Stitch with Next.js 14 on Vercel, which is production-proven, serverless-native, and fully compatible with Supabase, RevenueCat, and Google OAuth.

The backend layer is Hono.js running on Cloudflare Workers — a typed, ultra-lightweight HTTP framework designed for edge runtimes. It is fully compatible with Supabase's service-role client, handles RevenueCat webhooks, executes business logic, and keeps every compute unit serverless. There is no always-on server, no Docker, and no DevOps overhead.

2. Full architecture overview

All traffic enters through the Next.js frontend on Vercel. Authenticated requests requiring server-side business logic are proxied to the Hono API on Cloudflare Workers. The Workers communicate with Supabase using the service-role key (bypassing RLS for privileged operations) and with RevenueCat's REST API for entitlement checks and webhook processing.

Request lifecycle end to end
1. Browser → Next.js frontend (Vercel CDN)
2. Auth check → Supabase Auth (Google OAuth PKCE) → session JWT stored in httpOnly cookie
3. Page render → Next.js Server Component reads Supabase directly via service client for SSR
4. Mutations / business logic → Next.js calls Hono API (api.futura.app) with Bearer JWT
5. Hono Worker → validates JWT via Supabase Auth → executes logic → reads/writes Supabase Postgres
6. Entitlement checks → Hono queries user_subscriptions table (synced by RC webhook)
7. RevenueCat events → Stripe webhook → RC → POST to /webhooks/revenuecat on Hono → upserts user_subscriptions
Environment separation
Vercel env vars (frontend-safe): NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, NEXT_PUBLIC_API_BASE_URL

Cloudflare Worker secrets (never exposed to browser): SUPABASE_SERVICE_ROLE_KEY, REVENUECAT_API_KEY, REVENUECAT_WEBHOOK_SECRET, JWT_SECRET

// src/middleware/auth.ts import { createMiddleware } from 'hono/factory' import { createClient } from '@supabase/supabase-js' export const authMiddleware = createMiddleware<{ Bindings: Env }>( async (c, next) => { const token = c.req.header('Authorization')?.replace('Bearer ', '') if (!token) return c.json({ error: 'unauthorized' }, 401) const supabase = createClient(c.env.SUPABASE_URL, c.env.SUPABASE_SERVICE_ROLE_KEY) const { data: { user }, error } = await supabase.auth.getUser(token) if (error || !user) return c.json({ error: 'invalid_token' }, 401) c.set('userId', user.id) c.set('userEmail', user.email) await next() } )

Pro entitlement middleware

// src/middleware/entitlement.ts import { createMiddleware } from 'hono/factory' import { getSupabase } from '../lib/supabase' export const requirePro = createMiddleware<{ Bindings: Env }>( async (c, next) => { const userId = c.get('userId') const supabase = getSupabase(c.env) const { data } = await supabase .from('user_subscriptions') .select('entitlement, expires_at') .eq('user_id', userId) .single() const isPro = data?.entitlement === 'pro' && (!data.expires_at || new Date(data.expires_at) > new Date()) if (!isPro) return c.json({ error: 'pro_required', upgrade_url: '/upgrade' }, 403) await next() } )

Example route — contributions

// src/routes/contributions.ts import { Hono } from 'hono' import { zValidator } from '@hono/zod-validator' import { z } from 'zod' import { getSupabase } from '../lib/supabase' const router = new Hono<{ Bindings: Env; Variables: { userId: string } }>() const contributionSchema = z.object({ amount: z.number().positive().max(100000), contribution_date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/), note: z.string().max(200).optional(), }) router.get('/', async (c) => { const supabase = getSupabase(c.env) const { data, error } = await supabase .from('contributions') .select('*') .eq('user_id', c.get('userId')) .order('contribution_date', { ascending: false }) .limit(50) if (error) return c.json({ error: error.message }, 500) return c.json({ contributions: data }) }) router.post('/', zValidator('json', contributionSchema), async (c) => { const supabase = getSupabase(c.env) const body = c.req.valid('json') const { data, error } = await supabase .from('contributions') .insert({ ...body, user_id: c.get('userId') }) .select() .single() if (error) return c.json({ error: error.message }, 500) return c.json({ contribution: data }, 201) }) export { router as contributionsRouter }

RevenueCat webhook handler

// src/routes/webhooks.ts — receives RC purchase/cancellation events import { Hono } from 'hono' import { createHmac } from 'node:crypto' import { getSupabase } from '../lib/supabase' const router = new Hono<{ Bindings: Env }>() router.post('/revenuecat', async (c) => { // Verify RC webhook signature const sig = c.req.header('X-RevenueCat-Signature') ?? '' const body = await c.req.text() const expected = createHmac('sha256', c.env.REVENUECAT_WEBHOOK_SECRET) .update(body).digest('hex') if (sig !== expected) return c.json({ error: 'invalid_signature' }, 401) const event = JSON.parse(body) const { event: { type, app_user_id, expiration_at_ms } } = event const entitlement = ['INITIAL_PURCHASE', 'RENEWAL', 'UNCANCELLATION'].includes(type) ? 'pro' : 'free' const expiresAt = expiration_at_ms ? new Date(expiration_at_ms).toISOString() : null const supabase = getSupabase(c.env) await supabase.from('user_subscriptions').upsert({ revenuecat_customer_id: app_user_id, entitlement, expires_at: expiresAt, updated_at: new Date().toISOString(), }, { onConflict: 'revenuecat_customer_id' }) return c.json({ received: true }) }) export { router as webhooksRouter }

Vercel Edge middleware — auth guard

// middleware.ts — runs at Vercel edge before every request import { createMiddlewareClient } from '@supabase/auth-helpers-nextjs' import { NextResponse } from 'next/server' import type { NextRequest } from 'next/server' const PROTECTED = ['/dashboard', '/onboarding', '/projection', '/learn', '/settings'] export async function middleware(req: NextRequest) { const res = NextResponse.next() const supabase = createMiddlewareClient({ req, res }) const { data: { session } } = await supabase.auth.getSession() const isProtected = PROTECTED.some(p => req.nextUrl.pathname.startsWith(p)) if (isProtected && !session) { return NextResponse.redirect(new URL('/', req.url)) } return res } export const config = { matcher: ['/((?!_next/static|_next/image|favicon).*)'] }

5. Authentication — Google OAuth via Supabase

OAuth flow PKCE + httpOnly cookies
1. User clicks "Sign in with Google" → supabase.auth.signInWithOAuth({ provider: 'google' })
2. Browser redirected to Google consent screen
3. Google returns auth code to /auth/callback
4. Next.js route handler exchanges code for session via supabase.auth.exchangeCodeForSession(code)
5. Supabase JWT stored in httpOnly cookie (not localStorage — XSS-safe)
6. Database trigger on_auth_user_created fires → inserts row in profiles and user_subscriptions (entitlement: 'free')
7. User redirected to /onboarding (first time) or /dashboard (returning)
-- Supabase: trigger to bootstrap new user CREATE OR REPLACE FUNCTION public.handle_new_user() RETURNS trigger LANGUAGE plpgsql SECURITY DEFINER AS $$ BEGIN INSERT INTO public.profiles (id, email, display_name, avatar_url) VALUES ( NEW.id, NEW.email, NEW.raw_user_meta_data->>'full_name', NEW.raw_user_meta_data->>'avatar_url' ); INSERT INTO public.user_subscriptions (user_id, entitlement) VALUES (NEW.id, 'free'); RETURN NEW; END; $$; CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();

Row Level Security policies

-- Applied to: profiles, user_goals, contributions, streaks ALTER TABLE contributions ENABLE ROW LEVEL SECURITY; CREATE POLICY "select_own" ON contributions FOR SELECT USING (user_id = auth.uid()); CREATE POLICY "insert_own" ON contributions FOR INSERT WITH CHECK (user_id = auth.uid()); CREATE POLICY "update_own" ON contributions FOR UPDATE USING (user_id = auth.uid()); -- user_subscriptions: NO client write access — service role only ALTER TABLE user_subscriptions ENABLE ROW LEVEL SECURITY; CREATE POLICY "select_own_sub" ON user_subscriptions FOR SELECT USING (user_id = auth.uid()); -- Hono Worker uses SUPABASE_SERVICE_ROLE_KEY → bypasses RLS for writes -- This prevents any client-side entitlement spoofing

7. RevenueCat integration

Entitlement flow purchase → webhook → Supabase
Purchase: Frontend links to RevenueCat's hosted web checkout for the Pro plan (Stripe under the hood). No PCI scope for Futura. RC creates a customer using the Supabase user UUID as app_user_id.

Post-purchase: RC fires INITIAL_PURCHASE webhook → Hono /webhooks/revenuecat → upserts user_subscriptions.entitlement = 'pro'.

Renewal / cancellation: RC fires RENEWAL or CANCELLATION → Hono updates entitlement + expires_at accordingly.

Gate enforcement: Hono's requirePro middleware reads user_subscriptions on every protected API call — no RC API call at request time, keeping latency low.
RC customer ID strategy
Use the Supabase auth.uid() UUID as the RevenueCat app_user_id. This creates a 1:1 mapping without any lookup table. When the Hono worker first touches RC for a user, it calls GET /v1/subscribers/{supabase_uid} — RC creates the customer if they don't exist.
