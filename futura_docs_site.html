1. Problem statement

Gen Z — broadly, adults aged 18–27 in 2025 — is the first generation that will overwhelmingly rely on personal savings rather than pensions for retirement. Yet the concept of "retirement" triggers disengagement rather than action. Three compounding forces drive this:

Temporal distance
40+ yrs
Average gap between a 22-year-old and retirement, making it psychologically remote
Competing urgency
67%
Of Gen Z cites rent, student debt, or living costs as their top financial priority
Literacy gap
2 in 5
Gen Z adults cannot define compound interest or explain how an index fund works
Retirement blindness
73%
Have no retirement savings target and have never used a retirement calculator

Existing tools (Vanguard, Fidelity, MoneySuperMarket) are built for 45-year-olds — dense, jargon-heavy, and visually austere. Futura's hypothesis: if retirement can be made tangible, social, and habit-forming, Gen Z will engage with it the same way they engage with Duolingo or fitness apps.

4. Core features

4.1 Free tier features

User story · Onboarding
As a new user, I want to sign up with my Google account in under 30 seconds so I don't abandon before I've started.
Acceptance: Google OAuth completes, Supabase user row created, onboarding wizard launched — all within one page navigation.
User story · Retirement goal wizard
As a first-time user, I want a 3-step wizard that asks me my age, target retirement age, and desired monthly income, so Futura can calculate my savings target instantly.
Acceptance: wizard stores goal to Supabase user_goals table; projection visible on dashboard on completion.
User story · Dashboard
As a returning user, I want to see my projected retirement pot, current savings, and contribution streak on a single screen so I feel motivated to keep going.
Acceptance: dashboard loads in <1.5s; streak counter updates on contribution log; projection recalculates on any change.
User story · Contribution logging
As a user, I want to log a contribution (manual entry) in under 10 seconds so that the habit feels frictionless.
Acceptance: one-tap quick-log from dashboard; confirmation animation plays; streak incremented.
User story · Financial literacy bites
As a financially inexperienced user, I want a daily 60-second "Finance Bite" card so I can learn one concept at a time without feeling overwhelmed.
Acceptance: one new card per day from content CMS; card marked as read in Supabase; streak bonus awarded on read.

4.2 Pro tier features (RevenueCat-gated)

User story · Scenario modelling
As a Pro user, I want to model "what if" scenarios — changing retirement age, contribution amount, or investment return — so I understand the impact of my choices.
Acceptance: sliders update projection in real-time; paywall shown on first tap to free users with RevenueCat offering.
User story · Asset allocation breakdown
As a Pro user, I want to see an interactive breakdown of where my money should be invested (stocks, bonds, cash) based on my age and risk profile.
Acceptance: allocation chart rendered from user profile; updates when user adjusts risk slider; gated behind Pro entitlement check.
User story · PDF retirement report
As a Pro user, I want to download a personalised retirement roadmap PDF I can share with a financial adviser.
Acceptance: PDF generated server-side with user data; RevenueCat entitlement checked before generation endpoint is called.

1. UX philosophy

Gen Z users make snap judgements — if Futura feels like a bank's website in the first 10 seconds, they will leave. The UX must feel more like a productivity or fitness app than a financial tool. Four principles govern every design decision:

Principle 1
Progress over perfection
Celebrate any action — logging £5, reading a Finance Bite, setting a goal. Never show empty states without a clear, low-friction next action.
Principle 2
Reduce dread, show hope
Frame data positively. "You're on track to retire with £380,000" not "You need to save £380,000." Language is warm, never alarming.
Principle 3
Instant gratification
Every input changes something on screen immediately — sliders animate in real-time, contributions trigger animations. The app rewards engagement in the moment.
Principle 4
Conversational, not clinical
Copy uses "you/your" throughout. Financial terms are followed by plain-language parentheticals. No acronyms without explanation on first use.

2. Visual design direction

Aesthetic: Bold editorial minimalism

Futura uses a dark base with high-contrast lime-green accents — energetic, contemporary, and the opposite of a bank. Typography is the primary design element. White space is used aggressively to prevent overwhelm.

1. Architecture overview

Request flow (unauthenticated → authenticated)
Browser → Google Stitch frontend → Supabase Auth (Google OAuth) → Supabase Postgres → Supabase Edge Functions (business logic) → RevenueCat API (entitlement check) → Response
Serverless deployment topology
All server-side logic lives in Supabase Edge Functions (Deno runtime) and/or Vercel Serverless Functions. No persistent server. Google Stitch handles frontend rendering and routing. Static assets served from CDN. No Docker required.

2. Authentication — Google OAuth via Supabase

Integration pattern Supabase Auth
Supabase Auth natively supports Google OAuth as a provider. No custom OAuth implementation is needed. Configure in Supabase dashboard: Auth → Providers → Google. Supply Google Cloud Console OAuth 2.0 credentials (Client ID + Secret). Supabase handles PKCE flow, token exchange, and session management.
// Google Stitch — sign in import { createClient } from '@supabase/supabase-js' const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY) async function signInWithGoogle() { const { error } = await supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: `${window.location.origin}/auth/callback` } }) } // /auth/callback route — exchange code for session const { data, error } = await supabase.auth.exchangeCodeForSession( new URL(location.href).searchParams.get('code') )

On first login, a Supabase Database Trigger (on_auth_user_created) inserts a row into the profiles table, seeding default values. The JWT issued by Supabase Auth is used for all subsequent API calls via the Authorization: Bearer header.

4. RevenueCat integration (web)

Entitlement gating flow
1. On sign-in, frontend calls /functions/v1/rc-identify Edge Function with Supabase JWT.
2. Edge Function calls RevenueCat REST API: POST /v1/subscribers/{user_id} to create/retrieve customer.
3. Edge Function upserts user_subscriptions with current entitlement status.
4. Frontend reads entitlement from Supabase — no direct RC SDK call from the browser for security.
5. RevenueCat webhook (INITIAL_PURCHASE, RENEWAL, CANCELLATION events) hits /functions/v1/rc-webhook, which updates user_subscriptions.
// Supabase Edge Function: entitlement check helper async function getUserEntitlement(userId: string): Promise<'free' | 'pro'> { const { data } = await supabase .from('user_subscriptions') .select('entitlement') .eq('user_id', userId) .single() return data?.entitlement ?? 'free' } // Gate a Pro feature export async function handler(req: Request) { const { userId } = await getAuthUser(req) const entitlement = await getUserEntitlement(userId) if (entitlement !== 'pro') { return new Response(JSON.stringify({ error: 'pro_required' }), { status: 403 }) } // ... Pro feature logic }

For web purchases, RevenueCat Web Billing (Stripe) is used. The purchase button on the frontend opens RevenueCat's hosted checkout — no PCI scope for Futura. After payment, RC fires the webhook within seconds, updating the user's entitlement in Supabase.

6. Projection engine

Retirement projections run client-side for instant interactivity. The formula uses standard future value with monthly compounding:

// Retirement projection — runs in browser (no server call needed) function projectRetirementPot(params: { currentAge: number retirementAge: number monthlyContribution: number annualReturn: number // default 0.07 (7% global equities estimate) existingPot?: number }): number { const months = (params.retirementAge - params.currentAge) * 12 const monthlyRate = params.annualReturn / 12 const futureValueContributions = params.monthlyContribution * ((Math.pow(1 + monthlyRate, months) - 1) / monthlyRate) const futureValueExisting = (params.existingPot ?? 0) * Math.pow(1 + monthlyRate, months) return Math.round(futureValueContributions + futureValueExisting) }

The 7% annual return default is sourced from long-run global equity index averages (MSCI World, 1970–2023). Users can adjust this in Pro mode. A disclaimer is shown on all projection screens: "Projections are illustrative only and not financial advice."

8. Security and Row Level Security

RLS policies (all tables)
Every table has RLS enabled. Users can only SELECT/INSERT/UPDATE rows where user_id = auth.uid(). The user_subscriptions table is additionally protected: only the service role (used by Edge Functions) can UPDATE entitlement — never the anon client. This prevents client-side entitlement spoofing.
-- Example RLS policy: contributions table ALTER TABLE contributions ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users can only see their own contributions" ON contributions FOR SELECT USING (user_id = auth.uid()); CREATE POLICY "Users can insert their own contributions" ON contributions FOR INSERT WITH CHECK (user_id = auth.uid());
